# AGENTS.md - AI Agent Guidelines for iso-3166-2.js

## Project Overview

This is an ISO 3166-2 subdivision lookup library for JavaScript/Node.js. It provides functions to query country subdivisions (states, provinces, regions, etc.) by their ISO 3166-2 codes.

## Repository Structure

```
├── country/            # CSV data files for each country's subdivisions
│   ├── XX.csv          # Active subdivision data (XX = ISO 3166-1 alpha-2 code)
│   └── XX.deprecated.csv  # Lists deprecated subdivision codes for country XX
├── codes.csv           # ISO 3166-1 alpha-3 to alpha-2 conversion table
├── parse.py            # Python script that parses CSVs into data.js
├── functions.js        # JavaScript API (subdivision, country, data, codes)
├── iso3166.min.js      # Built output (generated by `make`)
├── Makefile            # Build process: parse.py → data.js → uglify → iso3166.min.js
├── test/index.js       # Test suite using tape
└── benchmark/index.js  # Performance benchmarks
```

## CSV Data Format

Each country CSV file (`country/XX.csv`) follows this format:

```
Country Name,Subdivision Code,Subdivision Name,Subdivision Type,Country Code
```

Example:
```
United States,US-CA,California,State,US
```

Deprecated CSV files (`country/XX.deprecated.csv`) contain only the deprecated subdivision codes, one per line.

## Naming Conventions for Subdivision Names

Subdivision names must use **internationally friendly English names** — the version most commonly recognized by international audiences. This means:

- **No local-language suffixes**: Use "Beijing" not "Beijing Shi", "Seoul" not "Seoul-teukbyeolsi"
- **Use English equivalents when well-known**: Use "Bavaria" not "Bayern", "Moscow" not "Moskva", "Cairo" not "Al Qāhirah"
- **Translate directional/descriptive terms**: Use "West Java" not "Jawa Barat", "North Khorasan" not "Khorāsān-e Shomālī"
- **Remove excessive transliteration diacritics**: Use "Tehran" not "Tehrān", "Baghdad" not "Baghdād", "Bihar" not "Bihār"
- **Keep diacritics that are part of the internationally used name**: Vietnamese (Hồ Chí Minh), French (Île-de-France), Portuguese (São Paulo) names retain their standard diacritics
- **Remove bracket annotations**: Use "Barcelona" not "Barcelona [Barcelona]"
- **Remove ceremonial/formal prefixes**: Use "Kuala Lumpur" not "Wilayah Persekutuan Kuala Lumpur"
- **Reference**: Cross-check with Wikipedia's ISO 3166-2 pages (e.g., https://en.wikipedia.org/wiki/ISO_3166-2:CN) for the commonly used English name

## Build Process

**You MUST rebuild `iso3166.min.js` every time CSV data is changed.** The built JS file is what consumers of the library actually use — CSV changes alone do nothing.

1. Edit CSV files in `country/` directory
2. Run `npm install` (first time only, to get `uglifyjs`)
3. Run `make` to rebuild `iso3166.min.js`
   - Requires Python 3 and `uglifyjs` (via `npm install`)
   - `parse.py` reads all CSVs and generates `data.js`
   - `uglifyjs` minifies `data.js` + `functions.js` into `iso3166.min.js`
4. Run tests with `npm test` to verify nothing is broken
5. Commit both the CSV changes and the updated `iso3166.min.js`

## Key Rules

- **Always regenerate `iso3166.min.js` after any CSV change** — run `make` every time you modify, add, or delete any file in `country/`. The library consumers import `iso3166.min.js`, so CSV edits have no effect until you rebuild.
- Never modify `iso3166.min.js` directly — it is a generated artifact
- When adding or updating subdivisions, only edit the CSV files
- When adding a new country, create a new `XX.csv` file following the existing format
- When deprecating codes, add them to the corresponding `XX.deprecated.csv` file
- Ensure subdivision codes match the ISO 3166-2 standard
- The `parse.py` script strips bracket annotations `[...]` from names automatically
- Tests in `test/index.js` verify specific subdivision lookups — update tests if changing names of tested subdivisions
- Commit both the CSV changes **and** the regenerated `iso3166.min.js` together
